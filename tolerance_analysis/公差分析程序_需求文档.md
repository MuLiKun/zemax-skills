# Zemax 公差分析自动化程序 —— 需求文档

> 版本：v0.4（高级 Excel 模式稳定；普通标准模板模式已具备后台/CLI/GUI 最小可用版）
> 目标：把"连接 Zemax → 读取镜头信息 → 用 Excel 或标准模板配置公差/评价函数/REPORT → 跑脚本式蒙特卡洛 → 保存 TSC/Worst/Best → 输出分项统计结果"的全流程固化为可复用程序，免去每次人工驱动。
> 使用说明（如何运行）见 `公差分析程序_使用说明.md`。

---

## 1. 项目目标与范围

### 1.1 目标
将一次完整的 Zemax 公差分析（基于本机 ZOS-API 2023 R1 已验证流程）做成**配置驱动**的自动化程序：高级用户可准备 zmx 文件 + Excel 配置表；普通用户可在 GUI/CLI 中选择标准模板、公差等级和 MC 参数，由程序自动生成兼容配置并完成公差分析。

### 1.2 范围（本期包含）
- 连接本机已打开的 OpticStudio（交互扩展）或独立实例。
- 读取镜头基本信息，辅助用户生成公差操作数。
- 以 Excel 驱动：公差操作数、评价函数、REPORT 分项、运行参数。
- 脚本式（TSC）蒙特卡洛公差分析。
- 保存 TSC、Worst Case、Best Case 文件。
- 输出各 REPORT 分项的统计结果，并导出统计 Excel。
- 支持独立分析已有 ZTD，统计文件输出到 ZTD 同目录。
- **PySide6 图形界面**（`gui.py`）：文件/参数选择、实时日志、运行计时、自动/独立 ZTD 统计、取消、关闭二次确认（已实现）。

### 1.3 不在本期范围（后续迭代）
- 灵敏度/反向求解模式（本期只做脚本式蒙特卡洛）。
- 自动出具 Word/PDF 报告。
- 分项直方图。

> **蒙特卡洛逐次进度（已确认无法实现）**：脚本式蒙卡（TSC，CriterionIndex=15）运行期间 ZOS-API 不暴露任何逐次进度——`Progress` 全程 0、结束才跳 100，无任何标量属性随运行变化；MC 样本文件也是**跑完才一次性写**（两次真机探测证实），且只写 NumberToSave 个。TSC 的 `SAVE` 命令只能存固定一个文件、不支持变量，无法逐圈保存（社区/官方实证）。因此 GUI 进度采用诚实方案：显示**运行总数 / 保存总数 + 已用时间计时器**，不做 1/N→N/N 伪进度。

## 3. 程序工作流（端到端）

```
[1] 解析命令行参数（zmx 路径、Excel 配置路径、输出目录、保存选项）
        │
[2] 连接 Zemax（ConnectAsExtension / CreateNewApplication），打开 zmx 副本
        │
[3] 读取镜头信息（面数、半径、厚度、玻璃、视场、波长、光阑面…）
    └─ 可选：导出"镜头信息"到 Excel，辅助用户填公差表
        │
[4] 读取 Excel 配置：
    ├─ 公差操作数（向导式 sheet + 逐行明细 sheet，合并）
    ├─ 评价函数（MFE 各行）
    ├─ REPORT 分项（标签 + 引用 MF 行号）
    └─ 运行参数（次数、补偿器、统计分布、保存选项…）
        │
[5] 写入 TDE（公差表）、写入/保存 MFE（评价函数 → .MF）
        │
[6] 生成 TSC 脚本（LOADMERIT + OPTIMIZE + 逐条 REPORT），保存到 Tolerance 目录
        │
[7] 配置 OpenTolerancing（脚本判据 CriterionIndex=15、补偿器、NumberOfRuns…）
    └─ 后台进程运行，轮询 IsRunning，保存 ZTD
        │
[8] 保存产物：TSC、Worst Case zmx、Best Case zmx
        │
[9] 读取 ZTD（ToleranceDataViewer.MonteCarloData），提取各 REPORT 分项
    └─ 优先按同名 TSC 的 REPORT 行反推统计 Excel 表头
        │
[10] 输出：统计 Excel（统计行 + MC 原始样本行）+ 日志
```

## 4. Excel 配置文件设计

### 4.1 总体约定
- **一个工作簿**包含全部配置，用 **sheet 名区分"示例"与"用户输入"**：
  - `示例_*` 前缀的 sheet：只读模板示例，**用户不要删除**，程序忽略其内容只作参考。
  - `输入_*` 前缀的 sheet：用户实际填写，程序读取这些 sheet。
- 程序首次运行可用 `--init-template` 生成一份带示例的空白工作簿。
- 表头行固定（第 1 行），数据从第 2 行起；空行跳过；`#` 开头的行视为注释。

### 4.2 Sheet 列表

| Sheet 名 | 用途 | 读/示例 |
|---|---|---|
| `说明` | 填写规则总说明 | 示例（只读） |
| `示例_公差向导` / `输入_公差向导` | 向导式公差（一类一行，批量生成） | 双份 |
| `示例_公差明细` / `输入_公差明细` | 逐行明细公差（一行一条，可直接粘贴 Zemax TDE 数据行） | 双份 |
| `示例_评价函数` / `输入_评价函数` | MFE 各行操作数 | 双份 |
| `示例_REPORT` / `输入_REPORT` | 最终 REPORT 分项 | 双份 |
| `示例_运行参数` / `输入_运行参数` | 次数/补偿器/分布/保存选项 | 双份 |

> 公差表同时支持**向导式**与**逐行明细**：先按 `输入_公差向导` 批量生成默认公差，再用 `输入_公差明细` 逐行追加。新模板已删除“操作”列，可直接粘贴 Zemax TDE 中间数据行；旧模板若仍保留“操作”列，程序继续兼容追加/覆盖/删除。

### 4.3 `公差向导` sheet（一类一行）

| 列 | 含义 | 示例值 | 备注 |
|---|---|---|---|
| 启用 | 是否启用该类 | Y / N | N 则跳过 |
| 公差类别 | 半径/曲率半径/厚度/面偏心X/面偏心Y/面倾斜X/面倾斜Y/元件偏心X/元件偏心Y/元件倾斜X/元件倾斜Y/面不规则/Zernike不规则度/折射率/阿贝% | 半径 | 中文类别名 |
| 数值 | 公差量级 | 3 | 单位随类别 |
| 单位 | 半径支持 光圈/fringe、毫米/mm、百分比/%；面倾斜X/Y支持 度、毫米；其他类别按 Zemax 向导含义填写 | 光圈 | 半径默认光圈，面倾斜默认度 |
| 起始面 | 应用起始面号 | 3 | |
| 结束面 | 应用结束面号 | 13 | |
| 跳过面 | 不应用的面（逗号分隔） | 9 | 如光阑面 |

### 4.4 `公差明细` sheet（一行一条，可粘贴 TDE 数据行）

| 列 | 含义 | 示例值 | 备注 |
|---|---|---|---|
| 操作数 | TDE 操作数代码 | TFRN | TFRN/TTHI/TEDX/TEDY/TETX/TETY/TIRR/TIND/TABB… |
| 面1 | Surf / Int1 | 5 | |
| 面2 | Surf2 / Int2 | 5 | 单面公差填同值或留空 |
| Min | 下限 | -3 | |
| Max | 上限 | 3 | |
| 注释 | 说明 | 半径3环 | 写入 TDE 注释列 |

新模板不再包含“操作”列，所有明细默认按追加处理，方便直接从 Zemax TDE 复制中间数据行粘贴到 Excel。旧模板如果仍保留“操作”列，程序仍兼容追加/覆盖/删除。

### 4.5 `评价函数` sheet（MFE 各行，对应 TSC 的 REPORT 行号）

| 列 | 含义 | 示例值 | 备注 |
|---|---|---|---|
| 行号 | MFE 行号 | 2 | 与 REPORT 的"MF行号"对应 |
| 操作数 | MF 操作数代码 | RSCE | RSCE/GENC/GMTT/GMTS… |
| Hx Hy Px Py | 视场/光瞳坐标 | 0 0 0 0 | |
| 参数1..6 | 操作数参数 | 34（lp/mm）等 | 随操作数含义不同 |
| 目标 | Target | 0 | |
| 权重 | Weight | 1 | 0 表示仅评估不优化 |
| 注释 | 说明 | 点列轴上 | |

> 程序按此 sheet 重建 MFE 并 `SaveMeritFunction()` 成 `.MF`，供 TSC `LOADMERIT`。

### 4.6 `REPORT` sheet（最终在结果中分项展示的项）

| 列 | 含义 | 示例值 | 备注 |
|---|---|---|---|
| 启用 | 是否报告 | Y | |
| 标签 | 结果列名 | SPOT_Hy0.0 | 出现在结果 Excel 列头 |
| MF行号 | 引用评价函数行号 | 2 | 对应 `评价函数` sheet 行号 |
| 方向 | 越小越好/越大越好 | 小 | 决定"最好/最差/2σ"取向 |
| 单位 | 显示单位 | mm | 仅说明 |

> 每个启用的 REPORT 行 → TSC 里一条 `REPORT "标签" 行号` → 蒙卡结果中一个独立分项列。

### 4.7 `运行参数` sheet（键值对）

| 参数键 | 含义 | 示例值 | 备注 |
|---|---|---|---|
| 蒙特卡洛次数 | NumberOfRuns | 200 | |
| 保存数量 | NumberToSave | 10 | 保存最差的前 N 个 |
| 统计分布 | 正态/均匀/抛物线 | 正态 | MonteCarloStatisticIndex |
| 补偿器模式 | 全部优化DLS/全部优化OD/无 | 无 | **总开关**：无=不补偿；DLS/OD=写优化行+补偿MF（双MF）。普通标准模板模式不再提供近轴焦点 |
| TSC优化周期 | TSC 内 OPTIMIZE n | 4 | |
| 中心波长号 | 评估用波长 | 2 或 0 | Wave 编号；标准模板模式默认 0，运行期自动使用 Zemax 主波长 |
| 后焦补偿面 | 在该面加 COMP 厚度补偿器；留空=不加 | 13 | 需模式≠无；COMP Param2=0 |
| 补偿Min | 补偿下限；留空=Zemax 自由调整 | （空）| COMP Min |
| 补偿Max | 补偿上限；留空=Zemax 自由调整 | （空）| COMP Max |
| 补偿线对 | 补偿专用 MF 的 GMTA 频率 lp/mm | 34 | 需模式≠无；GMTA P4 |
| 保存TSC | 是否保存 TSC | Y | 脚本式蒙卡必需，恒为 Y |
| 保存WorstCase | 是否保存最差 case | Y | **用户可配 Y/N** |
| 保存BestCase | 是否保存最好 case | Y | **用户可配 Y/N**；与 Worst 共用一个 API 开关 |
| 输出统计Excel | 是否输出结果统计 Excel | Y | 模板默认开启；统计文件输出到 ZTD 同目录 |
| 输出直方图 | 是否输出分项直方图PNG | N | **本期预留，默认关** |

## 5. 输出产物

用户选择的 `--outdir` / GUI 输出目录是**父输出目录**；每次运行会在其下创建独立结果目录：`公差分析_<镜头名>_YYYYMMDD_HHMMSS`。未指定输出目录时，父输出目录默认为 zmx 同级目录。

| 产物 | 路径/命名 | 触发条件 |
|---|---|---|
| 工作副本 | `<结果目录>\<zmx名>_tol.zmx` | 始终（不改原文件）|
| Excel 配置快照 | `<结果目录>\used_excel.xlsx` | 始终；记录运行期解析后的实际配置（含自动主波长、结束面、补偿面等） |
| 运行配置快照 | `<结果目录>\run_config.json` | 始终 |
| 运行日志 | `<结果目录>\run.log` + GUI 日志窗口 / 控制台输出 | 始终 |
| 视场映射报告 | `<结果目录>\field_mapping.txt` | 启用视场映射时输出；标准模板模式默认启用目标 `0,0.5,0.9`；仅记录匹配、插入与改写摘要 |
| 评价函数 | `<DataDir>\MeritFunction\<名>.MF` | 始终 |
| TSC 脚本 | `<DataDir>\Tolerance\<名>.TSC` | 始终；ZTD 统计用同名 TSC 反推表头 |
| 公差数据 | `<结果目录>\<名>.ZTD` | 始终（SaveTolDataFile=True）|
| Worst/Best Case | Zemax 按 `IsSaveBestWorstUsed + FilePrefix` 输出，运行后回读路径 | 保存WorstCase 或 保存BestCase 任一为 Y |
| 统计结果 Excel | ZTD 同目录，`<ZTD文件名>_统计.xlsx` | `输出统计Excel=Y` 或 GUI 独立分析已有 ZTD |
| 分项直方图 | `<结果目录>\hist\<标签>.png` | **本期预留，默认关**（后续迭代）|

### 5.1 统计结果内容
统计 Excel 采用“列为 REPORT/ZTD 指标、行为统计项与 MC 原始样本”的布局。固定统计行包括：`Cpk1.33下限`、`Cpk1.33上限`、`最大值`、`最小值`、`平均值`、`标准差`、`中位数`；其后为 `MC001`、`MC002` 等原始样本行。`Cpk1.33下限/上限` 对所有 REPORT 指标均按 `均值 ± 3*Cpk*标准差` 双边输出，并按方向标黄用户关注侧：越小越好标黄上限，越大越好标黄下限。

统计列包含自定义脚本总值、TSC `REPORT` 分项，以及 TDE 中的 `COMP` 补偿器项。REPORT 表头优先由同名 `.TSC` 反推：解析 `REPORT "标签" 行号` 为 `标签 (行号)`。COMP 不靠列名模糊匹配，而是按 `run_config.json` 中记录的 TDE 顺序定位。若未找到同名 TSC 或部分列无法识别，Excel 仍会生成，未识别列保留为 `colN`，并在日志/提示中列出搜索结果。

## 6. 运行入口（已实现）

当前保留两个入口，共享同一套核心逻辑（`toltool/pipeline.py`），详见 `公差分析程序_使用说明.md`：

| 入口 | 形态 | 自动跑蒙卡 | 读 ZTD |
|---|---|---|---|
| `gui.py` | PySide6 图形界面（暗色，实时日志+计时，推荐）| 是 | 是（自动/独立分析） |
| `tol_run.py` | 完整命令行（含统计输出）| 是 | 是（`--no-read` 关）|

`tol_run.py` 命令行：

普通标准模板模式（不需要手写 Excel）：

```powershell
python tol_run.py --standard --zmx "镜头.zmx" --outdir "输出目录" --connect standalone --standard-template 快速摸底 --tolerance-level 标准 --num-runs 20 --num-to-save 0
```

标准模板配置集中在 `toltool/standard_templates.py`，当前模板为临时标准；默认中心波长号为 0（运行期自动取主波长），结束面为 0（运行期自动取像面前一面），视场映射默认开启并使用 `0,0.5,0.9`。

高级 Excel 模式：

```
python tol_run.py \
    --zmx       "F:\个人文件\P3111\05304_tol.zmx" \
    --config    "tol_config_05304.xlsx" \
    --outdir    "F:\个人文件\P3111\tol_out" \  # 可选，默认 = zmx 同级目录
    [--init-template]      # 仅生成带示例的空白配置模板后退出
    [--connect extension|standalone]   # 默认 standalone（推荐）
    [--read-only]          # 只读镜头信息并打印，不跑分析
    [--validate-only]      # 只校验 zmx 路径与 Excel 配置，不连接 Zemax、不跑分析
    [--no-read]            # 跑完蒙卡即停，不读 ZTD
```

## 7. 模块划分（实际）

| 模块 | 职责 |
|---|---|
| `pipeline.py` | **单一来源**：`prepare_session()`（连接→TDE→MFE→TSC→Save）+ `run_montecarlo()`，四个入口共享 |
| `zos_connect.py` | 连接 Zemax、打开/另存副本、读取镜头信息 |
| `excel_io.py` | 读配置 sheet、生成示例模板 |
| `tde_builder.py` | 向导式 + 明细 → 写 TDE 公差表 |
| `mfe_builder.py` | 评价函数 sheet → 写 MFE → 存 .MF |
| `tsc_builder.py` | REPORT sheet → 生成 .TSC |
| `tol_runner.py` | 配置 OpenTolerancing、运行、保存 ZTD/Worst/Best。**运行逻辑与界面解耦：核心 `run(..., progress_cb=None, cancel_flag=None)` 既能被命令行调用，也能被 GUI 前台线程调用并通过 `progress_cb` 回调进度、`cancel_flag` 支持取消** |
| `ztd_reader.py` | 读 ZTD MonteCarloData → TSC 反推表头 → 统计并导出 Excel（含已知陷阱规避）|
| `gui.py` / `tol_run.py` | 图形界面与命令行入口，复用 `pipeline` |

## 8. 关键实现注意（来自实测，必须遵守）

1. **脚本式蒙卡 = CriterionIndex=15 + TSC**，每个 `REPORT` 行成为独立结果列（实现分项展示的核心）。
2. **`SaveTolDataFile=True` 必须显式设**（默认 False），并设 `TolDataFile`。
3. **长运行用后台进程**（Start-Process / subprocess），非阻塞 `Run()` + 轮询 `IsRunning`，避免被沙箱超时 kill。
4. **读 ZTD MonteCarloData 的陷阱**：循环变量与边界变量不能同名（PowerShell 大小写不敏感坑；Python 无此问题但要注意硬编码 `nrow=NumberOfRuns`）；不预读 `Values`，行优先连续读满所有列。
5. **ZTD 表头来源**：`MonteCarloData.Values` 不保证暴露 GUI 表头；优先用同名 TSC 的 `REPORT "标签" 行号` 反推为 `标签 (行号)`，找不到时保留 `colN` 兜底。
6. **脚本补偿判定**：脚本式蒙卡下 `CriterionCompIndex` 可能回读为“无”，不代表补偿未生效；实际补偿由 TSC 中的 `FASTFOCUS` / `OPTIMIZE n` / `OPTIMIZE n OD` 决定。
7. **公差运行诊断**：运行前后记录 Zemax 公差工具关键字段，失败信息中保留 `Succeeded`、`ErrorMessage`、`NumberToSave`、`IsSaveBestWorstUsed`、`SaveTolDataFile`、`TolDataFile` 等状态，便于判断失败发生在 Zemax 求解、样本保存还是 ZTD 保存阶段。
8. **中文路径**：Python 下注意以 UTF-8 处理；调用 DLL 时路径传字符串即可。
9. **不覆盖原始 zmx**，统一在副本上操作。
10. 工具一次只开一个：读 ZTD 前先 `tool.Close()`，再 `OpenToleranceDataViewer()`。

## 10. 已确认决策

1. **输出目录**：默认放在 **zmx 文件同级目录**；用户也可用命令行 `--outdir` 指定其他地址（不指定时取 zmx 同级）。
2. **「向导式」与「明细」同时使用时**：新模板的明细行会在向导结果之后逐行追加，便于从 Zemax TDE 复制数据行；旧模板若保留“操作”列，仍可执行覆盖/追加/删除。
3. **统计 Excel**：已实现并默认开启；列为 REPORT/ZTD 指标，行为 Cpk1.33 上下限、最大/最小/平均/标准差/中位数及 MC 原始样本；Cpk 双边输出并按方向标黄关注侧。直方图仍为后续迭代。
4. **GUI 交互**：已实现自动公差分析、自动统计 Excel、独立分析已有 ZTD、运行日志、计时、任务互斥、关闭保护，并记住上次文件/目录/连接模式。
5. **逐项 MC 文件导出取消**：原设想的「保存逐项MC」（每次试验各导出一个 zmx）当前 ZOS-API 在脚本式蒙卡下无对应开关，且占空间，已移除该参数。需要时再单独评估。
6. **Worst/Best 共用开关**：Zemax 的 `IsSaveBestWorstUsed` 是单一开关，无法分别只存 Worst 或只存 Best。程序逻辑为：保存WorstCase 或 保存BestCase 任一为 Y 即开启，两者皆 N 才关闭。
