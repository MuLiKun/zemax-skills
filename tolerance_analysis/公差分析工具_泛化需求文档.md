# Zemax 公差分析工具泛用化需求与架构设计

## 1. 背景与目标

现有公差分析工具已经具备完整的自动化能力：连接 Zemax、写入 TDE、公差评价函数生成、TSC 生成、Monte Carlo 运行、ZTD 读取与统计 Excel 导出。

但当前版本主要面向熟悉 Zemax 公差、MFE、TSC 和 Excel 配置结构的高级用户。若封装为 EXE 给更多同事使用，主要风险是：

1. Excel 配置项多，填写复杂，容易因面号、波长号、MFE 行号、REPORT 行号、补偿器参数错误导致失败；
2. 不同镜头结构差异较大，固定 Excel 模板难以覆盖所有场景；
3. 输出文件多，用户不容易判断哪些是核心结果；
4. Zemax API 出错信息不够直观，普通用户难以自查；
5. 当前流程更偏“专家工具”，还不是“通用分析产品”。

本设计目标是：在保留现有高级能力的基础上，新增更泛用、更易用、更稳定的普通分析模式，使不同同事可以在不了解复杂 Excel 的情况下完成标准公差分析。

---

## 2. 设计原则

### 2.1 泛用性优先

工具应适用于不同镜头，而不是只针对单个 RX 镜头或单个项目。不同镜头至少支持三类入口：

1. 使用工具内置标准公差模板；
2. 使用 Zemax 文件中已有的 TDE/MFE 设置；
3. 使用高级 Excel 完全自定义。

### 2.2 简单模式默认可用

普通用户不应被要求理解所有 Zemax 公差操作数。默认流程应是：

```text
选择镜头文件 → 选择分析模式 → 设置蒙卡次数 → 开始分析 → 获得统计 Excel
```

### 2.3 专家能力保留

现有 Excel 高级配置能力不能删除。它适合复杂项目、特殊评价函数、定制 TSC、特殊补偿器与调试。

### 2.4 原始镜头保护

所有分析必须基于工作副本运行，不直接修改用户原始 zmx 文件。

### 2.5 结果可追溯

每次运行应生成独立结果目录，包含工作副本、TSC、MF、ZTD、统计 Excel、日志和运行配置快照。

---

## 3. 用户角色

### 3.1 普通使用者

特点：

- 熟悉 Zemax 基本操作；
- 不熟悉 TDE/MFE/TSC 细节；
- 只希望快速获取公差统计结果。

推荐模式：标准模板模式、使用 Zemax 当前设置模式。

### 3.2 项目负责人 / 光学专家

特点：

- 能定义评价函数；
- 能判断补偿器、REPORT、MFE 行号；
- 需要控制公差策略。

推荐模式：高级 Excel 模式、模板维护模式。

### 3.3 工具维护者

特点：

- 维护 EXE、模板、默认参数；
- 处理 Zemax API 兼容性问题；
- 需要日志和调试入口。

推荐模式：调试模式、手动 TSC 模式。

## 4. 推荐功能分层

最终工具建议分为三种模式。

## 4.1 标准模板模式

面向普通同事。

用户只需要选择：

- 镜头文件；
- 输出目录；
- 公差等级；
- 评价指标；
- Monte Carlo 次数档位；
- 是否保存 Worst/Best 案例。

工具自动完成：

- 读取镜头面数、光阑面、波长、视场；
- 自动生成 TDE；
- 自动生成 MFE；
- 自动生成 TSC；
- 运行 Monte Carlo；
- 导出统计 Excel。

建议内置模板：

| 模板 | 用途 | 默认评价 |
|---|---|---|
| RX 标准分析 | 常规 RX 镜头快速评估 | Spot / GENC / GMTF |
| MTF 优先 | 关注 MTF 稳定性 | GMTT / GMTS |
| 点列优先 | 关注光斑变化 | RSCE |
| 能量集中度 | 关注 encircled energy | GENC |
| 快速摸底 | 少量蒙卡快速检查 | 少量指标 |

公差等级建议：

| 等级 | 用途 | 示例 |
|---|---|---|
| 宽松 | 初步趋势判断 | 半径/厚度/偏心/倾斜取较大值 |
| 标准 | 默认推荐 | 项目通用默认值 |
| 严格 | 量产风险评估 | 公差范围收紧 |

## 4.2 使用 Zemax 当前设置模式

面向已经在 Zemax 文件中设置好 TDE/MFE 的用户。

核心思想：

> 不强制用户填写 Excel。如果用户已经在 zmx 中配置好了公差数据和评价函数，则工具直接复用 Zemax 文件里的设置。

该模式建议支持两种子模式：

### 子模式 A：复用当前 TDE + 当前 MFE

流程：

```text
打开工作副本
不清空 TDE
不重建 MFE
基于当前 MFE 自动生成 TSC REPORT
运行公差
导出统计
```

适合：用户已经在 Zemax 中手工配置好公差和 Merit Function。

需要用户选择：

- REPORT 输出哪些 MFE 行；
- 或默认输出所有非空权重行；
- Monte Carlo 次数；
- 补偿器模式。

### 子模式 B：复用当前 TDE + 标准评价模板

流程：

```text
保留 zmx 内已有 TDE
工具生成标准 MFE/TSC
运行公差
导出统计
```

适合：用户已经配置公差，但评价函数希望用统一标准。

## 4.3 高级 Excel 模式

保留现有完整 Excel 配置。

适合：

- 特殊公差操作数；
- 特殊评价函数；
- 特殊 REPORT；
- 特殊补偿器；
- 项目专家自定义。

现有 Excel 模式应继续保留，但在 GUI 中标记为“高级”。

### 4.3.1 公差模板识别模式（低优先级）

在高级 Excel 模式下，后续可新增“公差模板识别模式”：

```text
结构同事提供公差文件/表格/文本 → 工具自动识别公差项 → 生成待确认结果 → 用户确认 → 写入 Excel 或 TDE
```

核心要求：

- 工具可以尝试从结构同事提供的公差资料中识别公差类型、面号、数值、单位和备注；
- 识别结果不能直接用于公差分析，必须先展示给用户确认；
- 推荐优先生成到 Excel 配置中，由用户检查后再运行；
- 如后续流程成熟，也可支持用户确认后直接写入 TDE；
- 对无法识别或置信度低的项目，应标记为“需人工确认”，不能静默跳过或自动猜测。

## 5.1 主界面布局

建议主界面分为四个区域：

```text

  分布:                [正态] [均匀] [抛物线]

四、结果与日志
  [开始分析]
  [停止]
  日志窗口
  [打开输出目录]
  [打开统计 Excel]
```



```text

[生成模板]

```

说明：识别结果必须经过用户确认后才能生成 Excel 或写入 TDE。

## 6. 架构设计

## 6.1 推荐总体架构

```text
GUI 层
  ├─ 普通模式界面
  ├─ Zemax 当前设置模式界面
  └─ 高级 Excel 模式界面

配置层
  ├─ 内置模板配置
  ├─ Zemax 当前文件读取配置
  ├─ Excel 配置读取
  └─ 公差资料识别与待确认配置

校验层
  ├─ 文件/目录校验
  ├─ 镜头结构校验
  ├─ 公差配置校验
  ├─ MFE/REPORT 校验
  └─ Zemax 环境校验

执行层
  ├─ 创建工作副本
  ├─ 写入或保留 TDE
  ├─ 写入或保留 MFE
  ├─ 生成 TSC
  ├─ 调用 Zemax 公差工具
  └─ 读取 ZTD

输出层
  ├─ 统计 Excel
  ├─ 日志文件
  ├─ 运行配置快照
  ├─ 工作副本
  └─ MC 案例文件
```

## 7. 配置模型设计

建议建立统一的内部配置对象。无论来自 GUI 模板、Zemax 当前文件、还是 Excel，最终都转换为同一结构。

```text
AnalysisConfig
  mode: standard_template | zemax_current | excel_advanced
  lens_file
  output_dir
  connect_mode
  tolerance_source
  merit_source
  report_source
  run_settings
  output_settings
```

### 7.1 公差来源 tolerance_source

```text
standard_template:
  tolerance_level
  categories
  start_surface
  end_surface
  skip_surfaces

zemax_current:
  keep_existing_tde = true

excel:
  tol_wizard
  tol_detail

recognized_tolerance_template:
  source_file
  parsed_items
  confidence
  require_user_confirm = true
```

### 7.2 评价函数来源 merit_source

```text
standard_template:
  metrics = [spot, genc, mtf]
  fields
  wavelength
  center_field_pointing      # 中心视场指向角 / 指向偏移
  fov                        # 视场角 FOV

zemax_current:
  keep_existing_mfe = true

excel:
  mfe_rows
```

### 7.3 REPORT 来源 report_source

```text
auto_from_mfe:
  include_weighted_rows = true
  include_comment_labels = true

standard_template:
  report_items

excel:
  report_rows
```

### 7.4 运行参数 run_settings

```text
num_runs
num_to_save
distribution
comp_mode
optimize_cycles
center_wave
save_best
save_worst
save_ztd
```

## 8. 推荐执行流程

## 8.1 标准模板模式流程

```text
1. 用户选择 zmx
2. 工具读取镜头信息
3. 自动推荐起始面/结束面/光阑面/主波长
4. 用户选择模板与公差等级
5. 工具生成内部配置
6. 创建工作副本
7. 清空并写入标准 TDE
8. 清空并写入标准 MFE
9. 生成 UTF-16 TSC
10. 运行 Monte Carlo
11. 读取 ZTD
12. 导出统计 Excel
13. 输出日志和配置快照
```

## 8.2 使用 Zemax 当前设置模式流程

```text
1. 用户选择 zmx
2. 创建工作副本
3. 读取工作副本中已有 TDE/MFE
4. 不覆盖已有 TDE/MFE
5. 自动生成 TSC，REPORT 来自当前 MFE 有效行
6. 运行 Monte Carlo
7. 导出统计 Excel
```

## 8.3 高级 Excel 模式流程

保持现有流程：

```text
读取 Excel
创建工作副本
写入 TDE
写入 MFE
生成 TSC
运行 Monte Carlo
导出统计 Excel
```

## 9. 关键稳定性改进

## 9.1 本地回滚机制

由于当前环境无法及时把修改推送到 GitHub 远程仓库，必须建立可靠的本地版本与回滚能力，避免代码或文档损坏后无法恢复。

要求：

```text
1. 每次重要修改前，自动或手动创建本地备份快照；
   - 备份目录建议：tolerance_analysis/_backups/YYYYMMDD_HHMMSS/
   - 备份内容：toltool/ 源码、gui.py、需求文档、build.ps1 等关键文件。
2. 使用本地 Git 仓库进行版本管理（即使无法 push 远程）：
   - 关键节点 git commit，保留可回退历史；
   - 网络恢复后再统一 push 到 GitHub。
3. 提供回滚操作说明：
   - 从 _backups 快照恢复；
   - 或使用 git checkout / git reset 回退到指定提交。
4. 备份目录不纳入 EXE 打包，仅用于开发与维护。
```

回滚优先级：本地 Git 历史为主，文件快照为兜底；两者并存以降低单点风险。

## 9.2 输出目录结构化

建议每次运行创建独立结果目录：

```text
输出目录/
  公差分析_镜头名_20260625_153000/
    work/
      xxx_tol.zmx
      xxx_tol.ZDA
    zemax/
      xxx_tol.ZTD
      MC cases...
    report/
      xxx_tol_统计.xlsx
    config/
      run_config.json
      used_template.json 或 used_excel.xlsx
    log/
      run.log
```

如果 Zemax 强制将 ZTD 保存到当前工作副本目录，则可以简化为：

```text
公差分析_镜头名_时间戳/
  xxx_tol.zmx
  xxx_tol.ZDA
  xxx_tol.ZTD
  xxx_tol_统计.xlsx
  run.log
  run_config.json
```

## 9.3 MC case 保存策略

最基础模式也允许保存 MC case，但需要做成明确可选项，避免用户不知情时生成大量文件。

建议默认策略：

```text
保存 MC case: 可选
保存数量: 用户可设置，默认 0 或少量
保存 Worst/Best: 可选
核心结果: 始终保存 ZTD + 统计 Excel
```

GUI 中建议给出提示：保存 MC case 会生成较多 zmx/ZDA 文件，适合需要追踪 Worst/Best 或复盘具体蒙卡样本时开启。

## 9.4 TSC 编码

TSC 文件统一使用 UTF-16 LE BOM，与 Zemax GUI 保存格式一致，避免中文 MF/TSC 文件名兼容问题。

## 9.5 运行前校验

正式运行前增加检查：

- zmx 是否存在；
- 输出目录是否可写；
- ZOS-API 是否可连接；
- 面号范围是否合法；
- 波长号是否合法；
- REPORT 引用的 MFE 行号是否存在；
- MFE 是否至少有一个有效评价行；
- TSC 是否可写入；
- 保存数量是否小于等于 Monte Carlo 次数；
- 当前模式下是否缺少必要字段。

## 9.6 测试用例开发与试用用户验证

工具改造过程中需要同步建立测试用例，用于验证代码稳定性、不同镜头适配性和 EXE 封装后的可用性。

建议测试用例分层：

```text
1. 基础代码测试
   - TSC UTF-16 写入与读取
   - 中文 zmx/MF/TSC/ZTD 路径兼容
   - Excel 配置读取
   - ZTD 统计读取与导出

2. 流程集成测试
   - 高级 Excel 模式完整跑通
   - 标准模板模式完整跑通
   - 使用 Zemax 当前 TDE/MFE 模式完整跑通
   - 保存 MC case / 不保存 MC case 两种输出策略

3. 镜头样本测试
   - 中文文件名镜头
   - 英文文件名镜头
   - 不同面数镜头
   - 不同视场/波长配置镜头
   - 已有 TDE/MFE 的镜头

4. 打包 EXE 测试
   - 新电脑或非开发环境运行
   - Zemax 连接检查
   - 输出目录权限检查
   - 错误提示是否普通用户能理解
```

建议试用用户由实际使用场景覆盖：

- 普通同事：只使用标准模板模式，验证是否无需理解 Excel 也能完成分析；
- 项目负责人 / 光学专家：使用 Zemax 当前设置模式和高级 Excel 模式，验证结果是否符合专业预期；
- 工具维护者：验证日志、备份、回滚、错误定位是否足够。

需要用户后续提供或确认的测试材料：

```text
1. 代表性 zmx 文件 2~5 个；
2. 至少 1 个中文路径或中文文件名样本；
3. 至少 1 个已有 TDE/MFE 的 Zemax 文件；
4. 期望输出指标，例如 Spot、MTF、GENC、指向偏移、FOV；
5. 是否需要保存 MC case，以及保存数量建议；
6. 参与试用的普通用户和专家用户名单或角色。
```

## 10. 错误提示设计

现有错误偏开发者。建议分为两层：

### 10.1 用户提示

例如：

```text
Zemax 无法计算名义值。
可能原因：
1. 当前评价函数在名义镜头上不能计算；
2. REPORT 引用的 MFE 行号不存在；
3. TSC 未正确加载 MF；
4. 补偿器设置不合理。

建议：
- 打开生成的工作副本；
- 在 Zemax 中手动更新 Merit Function；
- 或改用标准模板模式重新运行。
```

### 10.2 调试日志

保留详细信息：

- `SetupModeIndex`
- `CriterionIndex`
- `CriterionScript`
- `NumberOfRuns`
- `TolDataFile`
- `Succeeded`
- `ErrorMessage`
- 已检查的 ZTD 路径

## 11. 标准模板建议

## 11.1 标准 TDE 模板

默认公差类别：

- 半径；
- 厚度；
- 面偏心 X/Y；
- 面倾斜 X/Y；
- 元件偏心 X/Y；
- 元件倾斜 X/Y；
- 面不规则；
- 折射率；
- 阿贝数。

不同公差等级只调整数值，不改变类别。

## 11.2 标准评价模板

建议支持：

- RSCE：点列；
- GENC：能量集中度；
- GMTT / GMTS：几何 MTF 子午/弧矢；
- 可选 DMTA/DMTS：衍射 MTF，作为高级模板。

### 11.2.1 中心视场指向角 / 指向偏移 与 FOV（待细化）

除成像质量类指标外，标准评价模板还需新增“指向/视场”相关评价项，用于评估镜头在公差扰动下的视轴稳定性与视场变化：

- 中心视场指向角 / 指向偏移：
  - 含义：中心视场（轴上视场）主光线在像面上的落点相对名义位置的偏移，或对应的指向角变化（视轴漂移）；
  - 用途：评估公差导致的视轴/瞄准方向偏移，对带瞄准、对准、装调需求的镜头很关键；
  - 候选 Zemax 评价操作数：可考虑 REAY/REAX（实际光线像面坐标）、CENX/CENY（质心坐标）等组合，或自定义中心视场主光线落点的偏移量；
  - 统计目标：偏移量的 RSS / 2σ / 最大值。

- FOV（视场角）：
  - 含义：在公差扰动下，边缘视场对应的实际视场角或视场范围变化；
  - 用途：评估畸变/放大率类公差对实际视场覆盖的影响；
  - 候选思路：基于边缘视场主光线像高与名义像高的比值变化，换算视场角变化。

说明：本节为占位与方向说明，具体操作数选择、行号布局、REPORT 命名与统计口径后续细化（可由项目专家补充确定）。

## 13. 推荐最终产品形态

最终 EXE 应支持三种主入口：

```text
1. 一键标准分析
   - 适合普通同事
   - 不需要 Excel
   - 自动生成公差、评价函数、TSC

2. 使用 Zemax 当前设置
   - 适合已在 Zemax 文件中设置好 TDE/MFE 的用户
   - 工具负责运行、保存和统计

3. 高级 Excel 分析
   - 适合专家用户
   - 保留当前最大灵活性
```

这样可以同时满足：

- 同事使用 bug 少；
- 操作更便捷；
- 适用于不同镜头；
- 高级场景仍可完全定制。
