# Zemax 公差分析工具泛用化需求与架构设计

## 1. 背景与目标

现有公差分析工具已经具备完整的自动化能力：连接 Zemax、写入 TDE、公差评价函数生成、TSC 生成、Monte Carlo 运行、ZTD 读取与统计 Excel 导出。

但当前版本主要面向熟悉 Zemax 公差、MFE、TSC 和 Excel 配置结构的高级用户。若封装为 EXE 给更多同事使用，主要风险是：

1. Excel 配置项多，填写复杂，容易因面号、波长号、MFE 行号、REPORT 行号、补偿器参数错误导致失败；
2. 不同镜头结构差异较大，固定 Excel 模板难以覆盖所有场景；
3. 输出文件多，用户不容易判断哪些是核心结果；
4. Zemax API 出错信息不够直观，普通用户难以自查；
5. 当前流程更偏“专家工具”，还不是“通用分析产品”。

本设计目标是：在保留现有高级能力的基础上，新增更泛用、更易用、更稳定的普通分析模式，使不同同事可以在不了解复杂 Excel 的情况下完成标准公差分析。

---

## 2. 设计原则

### 2.1 泛用性优先

工具应适用于不同镜头，而不是只针对单个 RX 镜头或单个项目。不同镜头至少支持三类入口：

1. 使用工具内置标准公差模板；
2. 使用 Zemax 文件中已有的 TDE/MFE 设置；
3. 使用高级 Excel 完全自定义。

### 2.2 简单模式默认可用

普通用户不应被要求理解所有 Zemax 公差操作数。默认流程应是：

```text
选择镜头文件 → 选择分析模式 → 设置蒙卡次数 → 开始分析 → 获得统计 Excel
```

### 2.3 专家能力保留

现有 Excel 高级配置能力不能删除。它适合复杂项目、特殊评价函数、定制 TSC、特殊补偿器与调试。

### 2.4 原始镜头保护

所有分析必须基于工作副本运行，不直接修改用户原始 zmx 文件。

### 2.5 结果可追溯

每次运行应生成独立结果目录，包含工作副本、TSC、MF、ZTD、统计 Excel、日志和运行配置快照。

---

## 3. 用户角色

### 3.1 普通使用者

特点：

- 熟悉 Zemax 基本操作；
- 不熟悉 TDE/MFE/TSC 细节；
- 只希望快速获取公差统计结果。

推荐模式：标准模板模式、使用 Zemax 当前设置模式。

### 3.2 项目负责人 / 光学专家

特点：

- 能定义评价函数；
- 能判断补偿器、REPORT、MFE 行号；
- 需要控制公差策略。

推荐模式：高级 Excel 模式、模板维护模式。

视场号匹配与插入功能建议主要开放给此角色：

- 项目负责人 / 光学专家可判断目标归一化视场是否应插入；
- 可确认插入后的评价函数、REPORT、统计口径是否符合项目要求；
- GUI 中建议作为高级 Excel 模式的可选检查项，默认提示但不静默修改。

### 3.3 工具维护者

特点：

- 维护 EXE、模板、默认参数；
- 处理 Zemax API 兼容性问题；
- 需要日志和调试入口。

推荐模式：调试模式、手动 TSC 模式。

工具维护者需要实现和维护视场匹配/插入逻辑，包括 ZOS-API 兼容性、工作副本保护、日志追踪、失败回滚与测试脚本。

---

## 4. 推荐功能分层

最终工具建议分为三种模式。

## 4.1 标准模板模式

面向普通同事。

用户只需要选择：

- 镜头文件；
- 输出目录；
- 公差等级；
- 评价指标；
- Monte Carlo 次数档位；
- 是否保存 Worst/Best 案例。

工具自动完成：

- 读取镜头面数、光阑面、主波长、视场；
- 标准模板生成阶段不解析 zmx 文本面号，运行期打开 Zemax 工作副本后自动确定结束面（默认像面前一面）；
- GUI 标准模板模式不再提供波长选择，默认使用 Zemax 主波长；
- 优化补偿模式下若未填写后焦补偿面，默认使用像面前一面；
- 标准模板视场映射默认开启，当前目标归一化视场为 `0,0.5,0.9`；
- 自动生成 TDE；
- 自动生成 MFE；
- 自动生成 TSC；
- 运行 Monte Carlo；
- 导出统计 Excel。

建议内置模板：

| 模板 | 用途 | 默认评价 |
|---|---|---|
| RX 标准分析 | 常规 RX 镜头快速评估 | Spot / GENC / GMTF |
| MTF 优先 | 关注 MTF 稳定性 | GMTT / GMTS |
| 点列优先 | 关注光斑变化 | RSCE |
| 能量集中度 | 关注 encircled energy | GENC |
| 快速摸底 | 少量蒙卡快速检查 | 少量指标 |

公差等级建议：

| 等级 | 用途 | 示例 |
|---|---|---|
| 宽松 | 初步趋势判断 | 半径/厚度/偏心/倾斜取较大值 |
| 标准 | 默认推荐 | 项目通用默认值 |
| 严格 | 量产风险评估 | 公差范围收紧 |

## 4.2 使用 Zemax 当前设置模式

面向已经在 Zemax 文件中设置好 TDE/MFE 的用户。

核心思想：

> 不强制用户填写 Excel。如果用户已经在 zmx 中配置好了公差数据和评价函数，则工具直接复用 Zemax 文件里的设置。

该模式建议支持两种子模式：

### 子模式 A：复用当前 TDE + 当前 MFE

流程：

```text
打开工作副本
不清空 TDE
不重建 MFE
基于当前 MFE 自动生成 TSC REPORT
运行公差
导出统计
```

适合：用户已经在 Zemax 中手工配置好公差和 Merit Function。

需要用户选择：

- REPORT 输出哪些 MFE 行；
- 或默认输出所有非空权重行；
- Monte Carlo 次数；
- 补偿器模式。

### 子模式 B：复用当前 TDE + 标准评价模板

流程：

```text
保留 zmx 内已有 TDE
工具生成标准 MFE/TSC
运行公差
导出统计
```

适合：用户已经配置公差，但评价函数希望用统一标准。

## 4.3 高级 Excel 模式

保留现有完整 Excel 配置。

适合：

- 特殊公差操作数；
- 特殊评价函数；
- 特殊 REPORT；
- 特殊补偿器；
- 项目专家自定义。

现有 Excel 模式应继续保留，但在 GUI 中标记为“高级”。

### 4.3.1 视场号与归一化视场映射辅助

高级 Excel 模式中，评价函数存在两类不同的视场填写方式：

```text
RSCE：Param4 使用归一化视场，例如 0、0.5、0.7、0.9。
GENC / GMTT / GMTS：Param3 使用 Zemax 视场号。
```

不同 zmx 文件的视场数量、视场值、视场顺序可能不同，直接复用固定 Excel 模板时，容易出现“归一化视场”和“视场号”无法对应的问题。

需求：

1. 工具应支持读取待分析 zmx 的视场数据；
2. 按视场绝对值从小到大排序，输出原始 Zemax 视场号、X/Y 视场值、视场绝对值、归一化视场；
3. 归一化视场应保留正负号，中心视场为 0；常见 Y 视场使用 `Y / max(|Y|)`，二维 X/Y 视场可使用视场矢量在最大视场方向上的投影归一化；
4. 视场排序规则为按归一化视场绝对值从小到大，且同一绝对值下负值在前、正值在后，例如 `0、-0.25、0.25、-0.5、0.5、-0.7、0.7、-0.9、0.9、-1、1`；
5. 默认目标归一化视场为 `0、-0.25、0.25、-0.5、0.5、-0.7、0.7、-0.9、0.9、-1、1`；
6. 对每个目标归一化视场，自动匹配最接近的已有 Zemax 视场号；
7. 若目标归一化视场与最近已有视场差距大于 `0.05`，应提示用户可在工作副本视场表中插入该目标视场；
8. 插入视场只允许作用于工作副本，不能直接修改原始 zmx；插入前必须展示拟插入的 X/Y 视场值、目标归一化视场、插入后的预计视场号，并由用户确认；
9. 对常见 Y 视场，插入值可按最大正负视场缩放得到，例如目标 `-0.7` 对应 `Y = -0.7 * max(|Y|)`；对 X/Y 二维视场，按最大视场矢量方向缩放得到 `X/Y = target * max_field_vector`；
10. 评价函数与 REPORT 必须使用同一套目标归一化视场映射：RSCE 仍填写归一化视场，GENC/GMTT/GMTS/GMTA 填写匹配或插入后的 Zemax 视场号；
11. REPORT 和统计 Excel 中原来按“视场号”命名的标签，应统一改为按“归一化视场”打印，例如 `F0`、`F-0.25`、`F0.25`、`F-0.5`，避免不同 zmx 文件视场号不同造成歧义；必要时可在备注或附加列中保留实际 Zemax 视场号；
12. 主流程接入后仍不回写原始 Excel，只在内存中改写本次运行的 MFE/REPORT 配置；`used_excel.xlsx` 保留用户原始输入，`mapped_excel.xlsx` 记录本次实际映射后的 MFE/REPORT；
13. Excel `输入_评价函数` 可新增可选列 `目标归一化视场`，作为光学专家覆盖项；旧列名 `归一化视场` 继续兼容；
14. `目标归一化视场` 为空时后台自动推断：RSCE 从 `Param4` 推断，GENC/GMTT/GMTS/GMTA 从原 `Param3` 视场号查原始 zmx 归一化视场，再吸附到最近标准目标归一化视场；
15. `输入_运行参数` 增加 `启用视场映射`、`视场插入策略`、`视场匹配阈值`、`目标归一化视场`、`目标视场来源策略`，默认关闭、插入策略默认禁用、来源策略默认自动推断；
16. 若启用且策略为自动插入，插入只作用于 tol 工作副本，插入后必须保存工作副本并重新读取视场，再生成最终映射；
17. run.log、run_config.json 应记录原始视场、插入视场、最终映射、MFE/REPORT 改写数量和 `mapped_excel.xlsx` 路径；GUI 日志应打印最终映射表，便于确认；
18. 后续可在 GUI 高级 Excel 模式中增加“读取视场映射”按钮，把映射表显示给用户，并提示应填写的 GENC/GMTT/GMTS/GMTA 视场号，必要时提供“插入缺失视场到工作副本”操作。

输出示例：

```text
目标归一化视场  推荐视场号  实际归一化视场  偏差    REPORT标签
0.0              1           0.0000          0.0000  F0
-0.25            2          -0.2480          0.0020  F-0.25
0.25             3           0.2480          0.0020  F0.25
-0.5             4          -0.5100          0.0100  F-0.5
0.5              5           0.5100          0.0100  F0.5
-0.7             -           -               0.0800  F-0.7（建议插入）
0.7              -           -               0.0800  F0.7（建议插入）
```

### 4.3.2 公差模板识别模式（低优先级）

在高级 Excel 模式下，后续可新增“公差模板识别模式”：

```text
结构同事提供公差文件/表格/文本 → 工具自动识别公差项 → 生成待确认结果 → 用户确认 → 写入 Excel 或 TDE
```

核心要求：

- 工具可以尝试从结构同事提供的公差资料中识别公差类型、面号、数值、单位和备注；
- 识别结果不能直接用于公差分析，必须先展示给用户确认；
- 推荐优先生成到 Excel 配置中，由用户检查后再运行；
- 如后续流程成熟，也可支持用户确认后直接写入 TDE；
- 对无法识别或置信度低的项目，应标记为“需人工确认”，不能静默跳过或自动猜测。

## 5. GUI 设计建议

## 5.1 主界面布局

建议主界面分为四个区域：

```text
一、输入文件
  镜头文件 zmx:        [选择]
  输出目录:            [默认自动生成]
  连接模式:            [Standalone 推荐 / GUI 交互扩展]

二、分析模式
  分析方式:            [标准模板] [使用 Zemax 当前设置] [高级 Excel]

三、运行参数
  Monte Carlo:         [快速 20] [标准 200] [完整 1000] [自定义]
  分布:                [正态] [均匀] [抛物线]
  补偿器:              [无] [全部优化 DLS] [全部优化 OD]
  保存案例:            [保存数量 N] [是否保存 Worst/Best]

四、结果与日志
  [开始分析]
  [停止]
  日志窗口
  [打开输出目录]
  [打开统计 Excel]
```

```text
分析模板: [快速摸底 / RX 标准分析 / MTF 优先 / 点列优先 / 能量集中度]

中心波长: 不在 GUI 中显示，运行期自动使用主波长
```

## 5.3 使用 Zemax 当前设置模式界面



## 6. 架构设计

## 6.1 推荐总体架构

```text
GUI 层
  ├─ 普通模式界面
  ├─ Zemax 当前设置模式界面
  └─ 高级 Excel 模式界面

配置层
  ├─ 内置模板配置
  ├─ Zemax 当前文件读取配置
  ├─ Excel 配置读取
  └─ 公差资料识别与待确认配置

校验层
  ├─ 文件/目录校验
  ├─ 镜头结构校验
  ├─ 公差配置校验
  ├─ MFE/REPORT 校验
  └─ Zemax 环境校验

执行层
  ├─ 创建工作副本
  ├─ 写入或保留 TDE
  ├─ 写入或保留 MFE
  ├─ 生成 TSC
  ├─ 调用 Zemax 公差工具
  └─ 读取 ZTD

输出层
  ├─ 统计 Excel
  ├─ 日志文件
  ├─ 运行配置快照
  ├─ 工作副本
  └─ MC 案例文件
```

## 7. 配置模型设计

建议建立统一的内部配置对象。无论来自 GUI 模板、Zemax 当前文件、还是 Excel，最终都转换为同一结构。

```text
AnalysisConfig
  mode: standard_template | zemax_current | excel_advanced
  lens_file
  output_dir
  connect_mode
  tolerance_source
  merit_source
  report_source
  run_settings
  output_settings
```

### 7.1 公差来源 tolerance_source

```text
standard_template:
  tolerance_level
  categories
  start_surface
  end_surface
  skip_surfaces

zemax_current:
  keep_existing_tde = true

excel:
  tol_wizard
  tol_detail

recognized_tolerance_template:
  source_file
  parsed_items
  confidence
  require_user_confirm = true
```

### 7.2 评价函数来源 merit_source

```text
standard_template:
  metrics = [spot, genc, mtf]
  fields
  wavelength
  center_field_pointing      # 中心视场指向角 / 指向偏移
  fov                        # 视场角 FOV
  field_mapping              # zmx 视场号 ↔ 归一化视场映射

zemax_current:
  keep_existing_mfe = true

excel:
  mfe_rows
  field_mapping_preview      # 高级 Excel 模式下用于辅助核对 RSCE/GENC/GMTT/GMTS 的视场填写
```

视场映射数据结构建议：

```text
FieldMappingItem
  field_no                  # Zemax 原始视场号，1-based
  x
  y
  field_abs                 # sqrt(x^2 + y^2)，用于辅助展示
  normalized                # 保留正负号的归一化视场，例如 -0.5、0.5

FieldTargetMatch
  target_normalized         # 默认 0、-0.25、0.25、-0.5、0.5、-0.7、0.7、-0.9、0.9、-1、1
  field_no
  actual_normalized
  delta
  need_insert               # delta > 0.05 时为 true
  suggested_x               # 建议插入视场 X，仅用于工作副本
  suggested_y               # 建议插入视场 Y，仅用于工作副本
  report_label              # 按归一化视场命名，例如 F-0.25、F0.25
```

该映射默认只作用于本次运行内存配置和 tol 工作副本，不回写原始 Excel。启用视场映射后，程序会输出 `used_excel.xlsx` 原始快照和 `mapped_excel.xlsx` 映射后快照，便于用户核对评价函数、REPORT 标签与最终 Zemax 视场号。若需要插入缺失视场，也必须只修改工作副本，并把变更写入运行日志和运行配置快照。

### 7.3 REPORT 来源 report_source


## 8. 推荐执行流程

## 8.1 标准模板模式流程

```text
1. 用户选择 zmx
2. 用户选择模板、公差等级、MC 次数、保存数量、补偿方式和是否保存 WC/BC
3. 工具生成临时 Excel 兼容配置（模板集中维护在 `toltool/standard_templates.py`，当前模板为临时标准）
4. 创建工作副本并读取镜头信息
5. 运行期自动使用主波长、自动确定结束面（像面前一面）
6. 优化补偿且后焦补偿面为空时，自动设置为像面前一面
7. 清空并写入标准 TDE
8. 清空并写入标准 MFE
9. 生成 UTF-16 TSC
10. 运行 Monte Carlo
11. 读取 ZTD
12. 导出统计 Excel
13. 输出日志和配置快照
```

## 8.2 使用 Zemax 当前设置模式流程

```text
1. 用户选择 zmx
2. 创建工作副本
3. 读取工作副本中已有 TDE/MFE
4. 不覆盖已有 TDE/MFE
5. 自动生成 TSC，REPORT 来自当前 MFE 有效行
6. 运行 Monte Carlo
7. 导出统计 Excel
```

## 8.3 高级 Excel 模式流程

保持现有流程：

```text
读取 Excel
创建工作副本
读取工作副本视场数据
按目标归一化视场匹配 Zemax 视场号
若偏差 > 0.05，则在 tol 工作副本中插入缺失目标视场
重新读取工作副本视场数据并生成最终映射
从“目标归一化视场”列、RSCE Param4 或 GENC/GMTT/GMTS/GMTA 原 Param3 自动推断每行目标视场
按最终映射写入/更新评价函数：RSCE 使用归一化视场，GENC/GMTT/GMTS/GMTA 使用视场号
按归一化视场标签生成 REPORT，例如 F0、F-0.25、F0.25
输出 used_excel.xlsx 实际运行配置快照与 mapped_excel.xlsx 映射后快照
写入 TDE
写入 MFE
生成 TSC
运行 Monte Carlo
导出统计 Excel
```

要求：

```text
1. 视场插入只能发生在 tol 工作副本，不能修改原始 zmx；
2. 插入后必须保存工作副本，再继续写入 TDE/MFE/TSC；
3. 评价函数和 REPORT 必须使用同一份最终视场映射；
4. run.log、run_config.json、统计 Excel 中应记录原始视场、插入视场、最终映射、REPORT 标签；
5. 若插入失败，应停止运行并提示用户，不允许继续用错误视场号跑公差。
```

## 9. 关键稳定性改进

## 9.1 本地回滚机制

由于当前环境无法及时把修改推送到 GitHub 远程仓库，必须建立可靠的本地版本与回滚能力，避免代码或文档损坏后无法恢复。

要求：

```text
1. 每次重要修改前，自动或手动创建本地备份快照；
   - 备份目录建议：tolerance_analysis/_backups/YYYYMMDD_HHMMSS/
   - 备份内容：toltool/ 源码、gui.py、需求文档、build.ps1 等关键文件。
2. 使用本地 Git 仓库进行版本管理（即使无法 push 远程）：
   - 关键节点 git commit，保留可回退历史；
   - 网络恢复后再统一 push 到 GitHub。
3. 提供回滚操作说明：
   - 从 _backups 快照恢复；
   - 或使用 git checkout / git reset 回退到指定提交。
4. 备份目录不纳入 EXE 打包，仅用于开发与维护。
```

回滚优先级：本地 Git 历史为主，文件快照为兜底；两者并存以降低单点风险。

## 9.2 输出目录结构化

## 9.3 MC case 保存策略

最基础模式也允许保存 MC case，但需要做成明确可选项，避免用户不知情时生成大量文件。

建议默认策略：

```text
保存 MC case: 可选
保存数量: 用户可设置，默认 0 或少量
保存 Worst/Best: 可选
核心结果: 始终保存 ZTD + 统计 Excel
```

GUI 中建议给出提示：保存 MC case 会生成较多 zmx/ZDA 文件，适合需要追踪 Worst/Best 或复盘具体蒙卡样本时开启。

## 9.4 TSC 编码

TSC 文件统一使用 UTF-16 LE BOM，与 Zemax GUI 保存格式一致，避免中文 MF/TSC 文件名兼容问题。

## 9.5 运行前校验

正式运行前增加检查：

- zmx 是否存在；
- 输出目录是否可写；
- ZOS-API 是否可连接；
- 面号范围是否合法；
- 波长号是否合法；
- REPORT 引用的 MFE 行号是否存在；
- MFE 是否至少有一个有效评价行；
- TSC 是否可写入；
- 保存数量是否小于等于 Monte Carlo 次数；
- 当前模式下是否缺少必要字段。

## 11. 标准模板建议

## 11.1 标准 TDE 模板

默认公差类别：

- 半径；
- 厚度；
- 面偏心 X/Y；
- 面倾斜 X/Y；
- 元件偏心 X/Y；
- 元件倾斜 X/Y；
- 面不规则；
- 折射率；
- 阿贝数。

不同公差等级只调整数值，不改变类别。

## 11.2 标准评价模板

建议支持：

- RSCE：点列；
- GENC：能量集中度；
- GMTT / GMTS：几何 MTF 子午/弧矢；
- 可选 DMTA/DMTS：衍射 MTF，作为高级模板。

### 11.2.1 中心视场指向角 / 指向偏移 与 FOV（待细化）

除成像质量类指标外，标准评价模板还需新增“指向/视场”相关评价项，用于评估镜头在公差扰动下的视轴稳定性与视场变化：

- 中心视场指向角 / 指向偏移：
  - 含义：中心视场（轴上视场）主光线在像面上的落点相对名义位置的偏移，或对应的指向角变化（视轴漂移）；
  - 用途：评估公差导致的视轴/瞄准方向偏移，对带瞄准、对准、装调需求的镜头很关键；
  - 候选 Zemax 评价操作数：可考虑 REAY/REAX（实际光线像面坐标）、CENX/CENY（质心坐标）等组合，或自定义中心视场主光线落点的偏移量；
  - 统计目标：偏移量的 RSS / 2σ / 最大值。

- FOV（视场角）：
  - 含义：在公差扰动下，边缘视场对应的实际视场角或视场范围变化；
  - 用途：评估畸变/放大率类公差对实际视场覆盖的影响；
  - 候选思路：基于边缘视场主光线像高与名义像高的比值变化，换算视场角变化。

说明：本节为占位与方向说明，具体操作数选择、行号布局、REPORT 命名与统计口径后续细化（可由项目专家补充确定）。

## 11.3 REPORT 自动命名

建议格式：

```text
SPOT_F{field}_W{wave}
GENC95_F{field}
GMTT{freq}_F{field}
GMTS{freq}_F{field}
POINTING_F0          # 中心视场指向偏移
FOV_EDGE             # 边缘视场角变化
```

## 12. 开发路线

本文档只记录功能路线与需求范围。当前开发进度、任务状态、测试脚本和回归记录单独维护在：

- [公差分析工具_开发进度与测试记录.md](公差分析工具_开发进度与测试记录.md)

## 第一阶段：稳定性修复

目标：让当前高级 Excel 模式更稳定，适合内部试用。

主要任务：

1. 保留 TSC UTF-16 修复；
2. 建立本地回滚机制；
3. 实现工作副本，避免修改原始 zmx；
4. 输出目录时间戳隔离；
5. 支持可选保存 MC case，并限制默认生成文件数量；
6. 增加运行前校验；
7. 增加基础测试用例与试用用户验证清单；
8. 保存 `run.log`；
9. 保存运行配置快照与 Excel 配置快照；
10. 自动统计 ZTD 时限制为自定义脚本 + REPORT 项。

## 第二阶段：普通模式

目标：不依赖复杂 Excel 即可运行。

任务：

1. 增加“标准模板模式”；
2. GUI 增加模板、公差等级、蒙卡档位；
3. 自动读取镜头信息并推荐参数；
4. 内置标准 TDE/MFE/REPORT 模板（含指向/FOV 评价项）；
5. 导出统计结果摘要。

## 第三阶段：Zemax 当前设置模式

目标：支持用户在 Zemax 文档中设置公差和评价函数。

任务：

1. 支持保留当前 TDE；
2. 支持保留当前 MFE；
3. 自动从当前 MFE 生成 REPORT；
4. GUI 支持选择 REPORT 行；
5. 支持只生成 TSC 后交给用户手动运行。

## 第四阶段：模板管理

目标：让不同项目可维护自己的模板。

任务：

1. 模板保存为 JSON；
2. GUI 可导入/导出模板；
3. 支持项目默认模板；
4. 支持模板版本号；
5. 支持模板说明。

## 第五阶段：公差模板识别（低优先级）

目标：辅助处理结构同事提供的公差资料，降低手动录入 Excel 的工作量。

任务：

1. 支持导入结构公差资料（Excel、CSV 或文本，具体格式后续确认）；
2. 自动识别公差类型、面号、数值、单位、备注；
3. 生成待确认识别结果表；
4. 用户确认后生成到高级 Excel 配置；
5. 可选支持用户确认后直接写入 TDE；
6. 对低置信度或无法识别项进行人工确认提示。

## 13. 推荐最终产品形态

最终 EXE 应支持三种主入口：

```text
1. 一键标准分析
   - 适合普通同事
   - 不需要 Excel
   - 自动生成公差、评价函数、TSC

2. 使用 Zemax 当前设置
   - 适合已在 Zemax 文件中设置好 TDE/MFE 的用户
   - 工具负责运行、保存和统计

3. 高级 Excel 分析
   - 适合专家用户
   - 保留当前最大灵活性
```

这样可以同时满足：

- 同事使用 bug 少；
- 操作更便捷；
- 适用于不同镜头；
- 高级场景仍可完全定制。

## 15. 总结

现有工具的核心算法链路可以继续保留，不建议推倒重写。

推荐演进方向是：

```text
保留高级 Excel 模式
+ 新增标准模板模式
+ 新增使用 Zemax 当前设置模式
+ 增强校验、输出整理和错误提示
```

这样既能保证专家用户的灵活性，也能让普通同事用更少操作完成稳定的公差分析。
