### 1.7.2 真实 Zemax 回归执行步骤

建议优先用 1 个已验证镜头做小样本烟测，再扩展到多个镜头。每轮测试建议 MC 次数设为 2~5，保存数量设为 0~2，确认流程稳定后再提高次数。

#### A. 测试前准备

1. 确认 Zemax OpticStudio 可正常启动；
2. 确认测试 zmx 可手动打开；
3. 准备一个输出父目录，并确认有写权限；
4. 准备高级 Excel 配置文件，或使用普通标准模板模式；
5. 关闭可能占用旧输出文件的 Excel / 文本编辑器；
6. 记录 Zemax 版本、镜头文件、连接模式、运行模式和 MC 次数。

#### B. GUI 烟测顺序

1. 启动 GUI；
2. 选择“普通标准模板”模式；
3. 选择测试 zmx、输出目录、standalone 连接；
4. 设置 MC=2，保存数量=0 或 1；
5. 运行并检查是否生成结果目录；
6. 检查结果目录至少包含：
   - 工作副本 `.zmx`；

   - `run_config.json`；
   - `run.log`；
   - `.TSC` / `.MF`；
   - `.ZTD`；
   - `_统计.xlsx`；
   - 启用视场映射时的 `field_mapping.txt`。

#### C. 三种主模式回归

1. 普通标准模板模式：验证 UUID 临时配置不覆盖、主波长/结束面/补偿面自动解析、视场映射输出；
2. 高级 Excel 模式：验证 Excel 配置读取、TDE/MFE/TSC 写入、REPORT 与统计 Excel 一致；
3. 使用 Zemax 当前设置模式：手动在 Zemax 中保留 TDE/MFE，运行后确认 TDE 未被清空，当前 MFE 有效行进入 REPORT。

#### D. 独立 ZTD 分析回归

1. 选择上一轮自动分析生成的 `.ZTD`；
2. 不重新运行 Zemax，只执行 ZTD 统计；
3. 检查 `_统计.xlsx` 是否生成；
4. 对比 REPORT 标签、统计项数量、COMP 项是否符合预期；
5. 记录实际使用的配置来源，后续用于评估是否需要开发“优先同目录 used_excel.xlsx”。

#### E. 异常路径回归

1. Excel 表头删除一列，应在读取阶段报“表头列数不足”；
2. 公差明细 `面1/面2/Min/Max` 填非数字，应显示具体字段和值；
3. TSC 文件被占用时，应回退保存为 `_api.TSC` 并在日志中提示；
4. ZTD 未找到时，日志应列出候选路径检查过程；
5. standalone 会话释放失败时，日志应记录警告而不是静默吞掉。

### 1.7.4 COMP 列偏移专项验证

2026-06-30 已使用 `RX20251216_70B_16_BAG封装.zmx` 进行一次真实 ZTD 对比验证：Zemax 公差结果中的 COMP 后焦补偿项与导出的统计 Excel 中 `后焦补偿（TDE105_col107）` 样本值一致，说明当前样本下 COMP 列定位符合预期。

当前结论：COMP 列定位不再作为本样本阻塞项，但建议至少再用 1~2 个包含 COMP 的不同镜头复核后再标记为完全关闭。

待确认公式：

```python
col = report_count + tde_index
```

或：

```python
col = 1 + report_count + tde_index
```

本次样本支持当前实现，暂不修改 COMP 列定位逻辑；后续如新样本发现偏移，再按真实矩阵修正。
