# Zemax 公差分析程序 —— 使用说明与运行逻辑

> 面向使用者：如何准备文件、如何运行、各入口的区别、程序内部做了什么。
> 配套需求详见 `公差分析程序_需求文档.md`；API 实测经验见 skill `zemax-tolerance-analysis`。

---

## 一、它能做什么

给定**一个 zmx 镜头文件** + **一份 Excel 配置表**，程序自动：

1. 连接 OpticStudio，把 zmx 另存为工作副本（不动原始文件）；
2. 按 Excel 重建公差表（TDE）、评价函数（MFE）、公差脚本（TSC）；
3. 跑脚本式蒙特卡洛公差分析；
4. 保存 ZTD 公差数据、Worst/Best Case 文件；
5. （可选）读取 ZTD，把点列 / GENC / 几何 MTF 等各 REPORT 分项独立成列做统计。

## 三、四个运行入口（按场景选）

所有入口共享同一套核心逻辑（`toltool/pipeline.py`），区别只在**入口形态**和**是否自动跑/读**。

| 入口 | 适用场景 | 地址来源 | 是否自动跑蒙卡 | 是否读 ZTD |
|---|---|---|---|---|
| `main.py` | 固定一个项目，反复跑 | **写在文件顶部常量** | 是 | 否 |
| `tol_run.py` | 完整命令行，含统计输出 | 命令行参数 | 是 | 是（`--no-read` 可关）|
| `方案A_全自动\run_auto.py` | 全自动，输出独立目录 | 命令行参数 | 是 | 否 |
| `方案B_手动观察\run_prepare.py` | 只准备，手动在 GUI 点 Run 观察 | 命令行参数 | 否 | 否 |

### 1) main.py —— 最省事（推荐日常用）

在 [main.py](file:///e:/traewokspace/agentstudy/main.py) 顶部改三个常量：

```python
ZMX_FILE = r"F:\个人文件\P3111\05304_tol.zmx"      # 待分析的 zmx
CONFIG_FILE = r"F:\个人文件\P3111\tol_config_05304.xlsx"   # Excel 配置
CONNECT_MODE = "extension"                          # extension 或 standalone
```

然后直接运行（无需参数）：

```powershell
..\.venv\Scripts\python.exe -u main.py
```

也可用命令行临时覆盖写死的地址：

```powershell
..\.venv\Scripts\python.exe -u main.py --zmx "..." --config "..."
```

输出落在 `output\` 目录，跑完不读 ZTD（在 OpticStudio 或 ZTD 里查看）。

### 2) tol_run.py —— 完整命令行（要看分项统计用它）

```powershell
# 先生成配置模板（首次）
..\.venv\Scripts\python.exe -u tol_run.py --init-template --config tol_config_模板.xlsx

# 只读镜头信息，辅助填公差表（不跑分析）
..\.venv\Scripts\python.exe -u tol_run.py --read-only --zmx "...\05304_tol.zmx"

# 完整跑：建表 → 跑蒙卡 → 读 ZTD 出统计
..\.venv\Scripts\python.exe -u tol_run.py --zmx "...\05304_tol.zmx" --config tol_config_05304.xlsx

# 只跑不读（在 OpticStudio 里看结果）
..\.venv\Scripts\python.exe -u tol_run.py --zmx "..." --config "..." --no-read
```

参数说明：

| 参数 | 含义 |
|---|---|
| `--zmx` | 待分析 zmx 路径 |
| `--config` | Excel 配置路径 |
| `--outdir` | 输出目录（默认 = zmx 同级目录）|
| `--connect` | `extension`（默认）/ `standalone` |
| `--init-template` | 生成带示例的空白配置模板后退出 |
| `--overwrite` | 生成模板时覆盖已存在文件 |
| `--read-only` | 只读镜头信息，不跑分析 |
| `--no-read` | 跑完蒙卡即停，不读 ZTD |

### 3) 方案A / 方案B —— 验证与对照

- **方案A**：和 `main.py` 行为一致，但用命令行传地址，输出落在 `方案A_全自动\output`。
  ```powershell
  ..\.venv\Scripts\python.exe -u "方案A_全自动\run_auto.py" --zmx "..." --config "tol_config_05304.xlsx"
  ```
- **方案B**：只准备 TDE/MFE/TSC 后**停下**，打印手动操作步骤。你在 OpticStudio 的公差分析对话框里 Criterion 选 User Script、选打印出的 TSC、设次数、点 Run，可看原生窗口逐次试验实时滚动。
  ```powershell
  ..\.venv\Scripts\python.exe -u "方案B_手动观察\run_prepare.py" --zmx "..." --config "tol_config_05304.xlsx"
  ```

## 四、Excel 配置怎么填

用 `--init-template` 生成的模板含成对 sheet：`示例_*`（只读参考，勿删）与 `输入_*`（你填的）。程序只读 `输入_*`。

| Sheet | 填什么 |
|---|---|
| `说明` | 填写规则总说明（只读）|
| `输入_公差向导` | 一类公差一行，批量生成（半径/厚度/偏心/倾斜…）|
| `输入_公差明细` | 逐条公差，相对向导做追加/覆盖/删除 |
| `输入_评价函数` | MFE 各行操作数（RSCE/GENC/GMTT/GMTS…），行号对应 REPORT |
| `输入_REPORT` | 最终分项：启用 / 标签 / 引用 MF 行号 / 方向 |
| `输入_运行参数` | 见下表 |

### 运行参数（键值对）

| 参数键 | 含义 | 示例 | 是否由 Excel 生效 |
|---|---|---|---|
| 蒙特卡洛次数 | NumberOfRuns | 200 | 是 |
| 保存数量 | NumberToSave，保存最差前 N 个 | 10 | 是 |
| 统计分布 | 正态 / 均匀 / 抛物线 | 正态 | 是 |
| 补偿器模式 | 近轴焦点 / 全部优化DLS / 无 / 全部优化OD | 近轴焦点 | 是（脚本模式下常被 TSC 的 OPTIMIZE 接管）|
| TSC优化周期 | TSC 内 `OPTIMIZE n` | 4 | 是 |
| 中心波长号 | 评估用 Wave 编号 | 2 | 是 |
| 保存TSC | 始终保存（脚本式蒙卡必需）| Y | 恒为 Y |
| 保存WorstCase | 是否保存最差案例 | Y | 是（Y/N）|
| 保存BestCase | 是否保存最佳案例 | Y | 是（Y/N）|
| 输出统计Excel | 本期预留，默认关 | N | 预留 |
| 输出直方图 | 本期预留，默认关 | N | 预留 |

> **重要**：Worst / Best 在 Zemax API 是**同一个开关**（`IsSaveBestWorstUsed`），无法单独控制。程序逻辑是：两者任一为 Y 即开启保存，两者都为 N 才关闭。
> **保存数量 ≠ 逐项导出**：保存数量是「在结果里保留最差的前 N 个样本」，不是把每次运行各导出一个文件。逐项导出文件功能本期已移除。

## 五、程序内部运行逻辑（端到端）

```
入口（main.py / tol_run.py / 方案A / 方案B）
   │  传入 zmx、config、outdir、connect
   ▼
pipeline.prepare_session()  ← 四个入口共享的「准备」单一来源
   ├─ excel_io.read_config()      读 Excel 各 输入_* sheet → cfg
   ├─ zos_connect.connect()       连接 Zemax（extension/standalone）
   ├─ zos_connect.open_as_copy()  另存工作副本（不动原文件）
   ├─ tde_builder.build_and_write()   向导 + 明细 → 写 TDE 公差表
   ├─ mfe_builder.build_and_save()    评价函数 sheet → 写 MFE → 存 .MF
   ├─ tsc_builder.build_and_write()   REPORT sheet → 生成 .TSC
   └─ sess.sys.Save()             保存副本
   ▼
返回 Prepared（含 sess/cfg/run_params/各产物路径/计数）
   │
   ├─ 方案B：到此停下，打印手动操作步骤 ──► 用户在 GUI 点 Run
   │
   ▼ （main.py / tol_run.py / 方案A）
pipeline.run_montecarlo(prep)
   ├─ make_runspec(prep)          按 run_params 生成 RunSpec
   └─ tol_runner.run()            配置 OpenTolerancing + 非阻塞 Run + 轮询
        ├─ CriterionIndex=15（自定义脚本）+ 定位 TSC 索引
        ├─ NumberOfRuns / NumberToSave / 分布 / 补偿器
        ├─ SaveTolDataFile=True + TolDataFile（保存 ZTD）
        ├─ save_best_worst → IsSaveBestWorstUsed + FilePrefix
        └─ 轮询 IsRunning，进度回调（"运行中…"只提示一次）
   ▼
返回 RunResult（succeeded / ztd_path / bestworst_folder…）
   │
   ▼ （仅 tol_run.py 默认，未加 --no-read）
ztd_reader.read_ztd()
   └─ OpenToleranceDataViewer 读 MonteCarloData → 各 REPORT 分项统计
   ▼
format_table() 打印：标签 | N | 均值 | 标准差 | 最好 | 最差 | 2σ
```

### 模块职责一览（`toltool/` 包）

| 模块 | 职责 |
|---|---|
| `pipeline.py` | **单一来源**：prepare_session + run_montecarlo，四个入口共享 |
| `zos_connect.py` | 连接 Zemax、另存副本、读镜头信息 |
| `excel_io.py` | 读配置、生成示例模板 |
| `tde_builder.py` | 向导 + 明细 → 写 TDE |
| `mfe_builder.py` | 评价函数 sheet → 写 MFE → 存 .MF |
| `tsc_builder.py` | REPORT sheet → 生成 .TSC（LOADMERIT + OPTIMIZE + 逐条 REPORT）|
| `tol_runner.py` | 配置并运行蒙卡（界面解耦：progress_cb 回调 + cancel_flag 取消）|
| `ztd_reader.py` | 读 ZTD MonteCarloData → 分项统计 |

### 文件调用顺序（一次完整运行）

```
入口文件（main.py / tol_run.py / run_auto.py / run_prepare.py）
   └─▶ toltool/pipeline.py : prepare_session()
          ├─▶ excel_io.py     read_config()        ① 读 Excel 各 输入_* sheet
          ├─▶ zos_connect.py  connect()/open_as_copy()  ② 连接 + 另存副本
          ├─▶ tde_builder.py  build_and_write()    ③ 写 TDE 公差表
          ├─▶ mfe_builder.py  build_and_save()     ④ 写 MFE → 存 .MF
          └─▶ tsc_builder.py  build_and_write()    ⑤ 生成 .TSC
                                                   （方案B 到此停下，手动跑）
   └─▶ toltool/pipeline.py : run_montecarlo()       ⑥ main.py/tol_run.py/方案A
          └─▶ tol_runner.py   run()                配置 OpenTolerancing + 跑蒙卡
   └─▶ toltool/ztd_reader.py : read_ztd()/format_table()  ⑦ 仅 tol_run.py 默认
```

> 一句话：**入口 → pipeline（依次调 excel_io / zos_connect / tde_builder / mfe_builder / tsc_builder）→ tol_runner 跑蒙卡 →（tol_run.py 再调）ztd_reader 出统计**。`__init__.py` 仅作包标识，不参与流程逻辑。

## 七、常见问题

- **连不上 / 连接报错**：多半是没进交互扩展，或上次连接已消耗等待状态 → 重新点一次「编程→交互扩展」。
- **写 TSC 报权限错误（PermissionError）**：用后台隐藏进程启动 python 会写不了 `Documents\Zemax\Tolerance`。**在你自己的终端前台运行即可**，这不是代码问题。
- **"运行中…" 只打印一次**：这是有意为之，蒙卡运行期间每 2 秒轮询一次，只在首次提示，避免刷屏。完成后会提示结果。
- **补偿器模式好像没生效**：脚本式蒙卡里补偿器优化由 TSC 的 `OPTIMIZE n` 接管，运行参数里的「补偿器模式」常被它覆盖，属正常。
- **改了 Excel 没效果**：确认填的是 `输入_*` sheet（不是 `示例_*`），且键名与上表一致。
