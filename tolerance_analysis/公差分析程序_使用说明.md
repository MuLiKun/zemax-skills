# Zemax 公差分析程序 —— 使用说明与运行逻辑

> 面向使用者：如何准备文件、如何运行、各入口的区别、程序内部做了什么。
> 配套需求详见 `公差分析程序_需求文档.md`；API 实测经验见 skill `zemax-tolerance-analysis`。

---

## 一、它能做什么

给定**一个 zmx 镜头文件** + **一份 Excel 配置表**，程序自动：

1. 连接 OpticStudio，把 zmx 另存为工作副本（不动原始文件）；
2. 按普通标准模板或 Excel 重建公差表（TDE）、评价函数（MFE）、公差脚本（TSC）；
3. 跑脚本式蒙特卡洛公差分析；
4. 保存 ZTD 公差数据、Worst/Best Case 文件；
5. （可选）读取 ZTD，把点列 / GENC / 几何 MTF 等各 REPORT 分项独立成列做统计。

## 二、运行前准备（每次必做）

> **运行环境（venv 与目录）**
> - 项目根目录：`e:\traewokspace\agentstudy\tolerance_analysis`
> - Python 解释器（venv 在 agent 工作区共享，位于项目上一级）：`e:\traewokspace\agentstudy\.venv\Scripts\python.exe`
> - 约定：先 `cd` 进项目根目录，再用相对路径 `..\.venv\Scripts\python.exe` 调用解释器（本文档命令均以此为前缀）。
> ```powershell
> cd e:\traewokspace\agentstudy\tolerance_analysis
> ..\.venv\Scripts\python.exe -V    # 验证解释器可用
> ```

1. 打开 OpticStudio，载入要分析的镜头。
2. 菜单 **编程 → 交互扩展（Interactive Extension）**，点击后程序进入「等待连接」状态。
   - ⚠️ 每次连接会消耗掉一次等待状态。**程序跑完或断开后，要再次运行必须重新点一次交互扩展。**
3. 若使用高级 Excel 模式，确认 Excel 配置已填好（见第四节）；若使用普通标准模板模式，不需要手写 Excel。
4. 在**你自己的终端前台**运行（不要用后台隐藏进程启动，否则写 TSC 会遇到权限问题）。先切到项目目录：
   ```powershell
   cd e:\traewokspace\agentstudy\tolerance_analysis
   ```

   后续命令中的 `..\.venv\Scripts\python.exe` 即工作区共享的 Python 解释器（venv 在上一级目录）。

> 也可用 `--connect standalone` 走独立实例（`CreateNewApplication()`），此时不需要交互扩展，但操作的是后台独立进程而非你看到的 GUI。

## 三、运行入口（按场景选）

当前保留两个入口，共享同一套核心逻辑（`toltool/pipeline.py`），区别只在**入口形态**。

| 入口 | 适用场景 | 地址来源 | 是否自动跑蒙卡 | 是否读 ZTD |
|---|---|---|---|---|
| `gui.py` | **图形界面，日常首选** | **窗口里选** | 是 | 是（可自动/独立分析） |
| `tol_run.py` | 完整命令行，含统计输出 | 命令行参数 | 是 | 是（`--no-read` 可关）|

### 0) gui.py —— 图形界面（推荐）

```powershell
.\.venv\Scripts\python.exe -u tolerance_analysis\gui.py
```

窗口里选 zmx、Excel 配置、输出目录与连接模式，点「开始分析」即可，日志实时回显在下方文本框。GUI 会记住上次文件、目录与连接模式。选择的输出目录是父目录；每次运行会自动创建 `公差分析_<镜头名>_YYYYMMDD_HHMMSS` 结果目录，工作副本、ZTD、统计 Excel、`run.log`、`run_config.json`、`used_excel.xlsx` 都放在该结果目录中；启用视场映射时额外输出轻量文本报告 `field_mapping.txt`。若配置 Excel 中 `输出统计Excel=Y`，跑完会自动读取 ZTD 并在 ZTD 同目录生成统计 Excel。也可以使用界面里的独立 ZTD 分析入口，选择已有 `.ZTD` 后单独导出统计 Excel。业务逻辑全部复用命令行同一套 `toltool.pipeline`，界面只负责交互与线程调度。

特性：

- **暗色主题**（含窗口与弹窗标题栏）。
- **实时日志**框 + 「复制日志」「清空日志」按钮。
- **运行计时器**：状态栏显示「已用时间 mm:ss」（每秒跳动）。
- **运行总数 / 保存总数**：状态栏显示「运行总数 N 次 / 保存总数 M 个」，完成后显示「已完成：运行 N 次 / 保存 M 个」。
  > 注意：这是**总数展示**，不是逐次进度——脚本式蒙卡运行期间 Zemax 不暴露逐次进度（详见第七节）。
- **取消**按钮：运行中可请求取消。
- **ZTD 统计 Excel**：自动分析和独立分析都会把统计文件输出到对应 ZTD 所在目录，命名为 `<ZTD文件名>_统计.xlsx`。统计行固定包含名义值、Cpk1.33 下限/上限、最大值、最小值、平均值、标准差、中位数；名义值在跑蒙特卡洛前直接从评价函数读取（随 `run_config.json` 的 `report_meta` 落盘），Cpk1.33 下限/上限双边输出，并按方向标黄用户关注侧（越小越好标黄上限，越大越好标黄下限）。
- **TSC 反推表头**：读取 ZTD 时优先查找同名 `.TSC`，解析 `REPORT "标签" 行号` 生成 Excel 列头；找不到同名 TSC 时不阻断导出，未识别列保留为 `colN`。
- **关闭保护**：运行中点关闭窗口会弹二次确认；确认后强制中断蒙特卡洛并关闭后台 Zemax 实例（Standalone 模式），未完成结果丢失。
- **普通标准模板模式**：模板内容集中在 `toltool/standard_templates.py`，当前为临时标准；GUI 不显示波长，运行期自动使用 Zemax 主波长；公差结束面自动取像面前一面；选择 DLS/OD 优化但未填后焦补偿面时，自动取像面前一面；标准模板视场映射默认开启，目标为 `0,0.5,0.9`；运行期临时配置生成 `<镜头名>_标准模板配置_<8位uuid>.xlsx`，避免覆盖用户导出的同名配置；「保存 WC/BC」勾选后会开启 Zemax Worst/Best 保存。

连接模式（窗口里选）：

- **Standalone（推荐）**：程序后台自起 Zemax 实例，跑完自动释放，不需要交互扩展。
- **GUI 模式**：连入已在 OpticStudio「交互扩展」等待的窗口，不关它。

### 1) tol_run.py —— 完整命令行（要看分项统计用它）

```powershell
# 先生成配置模板（首次）
..\.venv\Scripts\python.exe -u tol_run.py --init-template --config tol_config_模板.xlsx

# 只读镜头信息，辅助填公差表（不跑分析）
..\.venv\Scripts\python.exe -u tol_run.py --read-only --zmx "...\05304_tol.zmx"

# 只校验 zmx 路径与 Excel 配置，不连接 Zemax、不跑分析
..\.venv\Scripts\python.exe -u tol_run.py --validate-only --zmx "...\05304_tol.zmx" --config tol_config_05304.xlsx

# 完整跑：建表 → 跑蒙卡 → 读 ZTD 出统计
..\.venv\Scripts\python.exe -u tol_run.py --zmx "...\05304_tol.zmx" --config tol_config_05304.xlsx

# 只跑不读（在 OpticStudio 里看结果）
..\.venv\Scripts\python.exe -u tol_run.py --zmx "..." --config "..." --no-read

# 普通标准模板模式（不需要手写 Excel）
..\.venv\Scripts\python.exe -u tol_run.py --standard --zmx "..." --outdir "..." --connect standalone --num-runs 20 --num-to-save 0

# 使用 Zemax 当前设置模式（复用 zmx 中已有 TDE/MFE，不需要 --config）
..\.venv\Scripts\python.exe -u tol_run.py --current-settings --zmx "..." --outdir "..." --connect standalone --num-runs 20 --comp-mode 全部优化DLS
```

参数说明：

| 参数 | 含义 |
|---|---|
| `--zmx` | 待分析 zmx 路径 |
| `--config` | Excel 配置路径；标准模板/当前设置模式可不填 |
| `--standard` | 普通标准模板模式，不需要手写 Excel |
| `--current-settings` | 使用 Zemax 当前设置模式，复用 zmx 中已有 TDE/MFE，不重建 TDE |
| `--outdir` | 父输出目录（默认 = zmx 同级目录）；实际产物进入其下的时间戳结果目录 |
| `--connect` | `extension`（默认）/ `standalone` |
| `--init-template` | 生成带示例的空白配置模板后退出 |
| `--overwrite` | 生成模板时覆盖已存在文件 |
| `--read-only` | 只读镜头信息，不跑分析 |
| `--validate-only` | 只校验 zmx 路径与 Excel 配置，不连接 Zemax、不跑分析 |
| `--no-read` | 跑完蒙卡即停，不读 ZTD |
| `--num-runs` / `--num-to-save` | 标准模板/当前设置模式的蒙特卡洛次数与 MC case 保存数量 |
| `--comp-mode` | 标准模板/当前设置模式的补偿器模式；当前设置模式只在 TDE 已有 `COMP` 时生效 |

## 四、Excel 配置怎么填

用 `--init-template` 生成的模板含成对 sheet：`示例_*`（只读参考，勿删）与 `输入_*`（你填的）。程序只读 `输入_*`。

| Sheet | 填什么 |
|---|---|
| `说明` | 填写规则总说明（只读）|
| `输入_公差向导` | 一类公差一行，批量生成（半径/厚度/偏心/倾斜…）；半径单位支持光圈/毫米/百分比且默认光圈，面倾斜支持度/毫米且默认度，支持 Zernike不规则度（需当前 Zemax API 暴露对应属性）|
| `输入_公差明细` | 逐条公差，可直接粘贴 Zemax TDE 中间数据行；新模板默认逐行追加 |
| `输入_评价函数` | MFE 各行操作数（RSCE/GENC/GMTT/GMTS…），行号对应 REPORT |
| `输入_REPORT` | 最终分项：启用 / 标签 / 引用 MF 行号 / 方向 |
| `输入_运行参数` | 见下表 |

### 运行参数（键值对）

| 参数键 | 含义 | 示例 | 是否由 Excel 生效 |
|---|---|---|---|
| 蒙特卡洛次数 | NumberOfRuns | 200 | 是 |
| 保存数量 | NumberToSave，保存最差前 N 个 | 10 | 是 |
| 统计分布 | 正态 / 均匀 / 抛物线 | 正态 | 是 |
| 补偿器模式 | 全部优化DLS / 全部优化OD / 无 | 无 | 是（**总开关**，见下）；普通标准模板模式不提供近轴焦点 |
| TSC优化周期 | TSC 内 `OPTIMIZE n` | 4 | 是 |
| 中心波长号 | 评估用 Wave 编号 | 2 或 0 | 是；标准模板模式默认 0，运行期自动使用主波长 |
| 后焦补偿面 | 在该面加 COMP 厚度补偿器；留空=不加 | 13 | 是（需模式≠无）|
| 补偿Min | 补偿下限；留空=Zemax 自由调整 | （空）| 是 |
| 补偿Max | 补偿上限；留空=Zemax 自由调整 | （空）| 是 |
| 补偿线对 | 补偿专用 MF 的 GMTA 频率 lp/mm | 34 | 是（需模式≠无）|
| 保存TSC | 始终保存（脚本式蒙卡必需）| Y | 恒为 Y |
| 保存WorstCase | 是否保存最差案例 | Y | 是（Y/N）|
| 保存BestCase | 是否保存最佳案例 | Y | 是（Y/N）|
| 输出统计Excel | 跑完后自动读取 ZTD 并导出统计 Excel | Y | 是，模板默认开启 |
| 输出直方图 | 本期预留，默认关 | N | 预留 |

> **补偿器模式 = 总开关**：
> - **无** → TSC 不写优化行、不生成补偿专用 MF、TDE 也不加 COMP（即使填了后焦补偿面也忽略）。
> - **全部优化DLS** → `OPTIMIZE n`；**全部优化OD** → `OPTIMIZE n OD`。
>   并生成补偿专用 MF（GMTA，频率=补偿线对，视场=1、目标=1、权重=1），TSC 走双 `LOADMERIT`（先补偿 MF + 优化行，再报告 MF + REPORT 各行）。普通标准模板模式不再提供近轴焦点。
> - 模式≠无 **且** 后焦补偿面>0 → 额外在 TDE 该面加 COMP 厚度补偿器（Param2=0，Min/Max 留空则 Zemax 自由调整）。
> - 高级 Excel 模式下，模式≠无 **但** 后焦补偿面留空 → 仍写优化行/补偿 MF，仅 TDE 不加 COMP（补偿由 TSC 负责，日志会提示）。普通标准模板模式下，DLS/OD 优化且后焦补偿面为空时会自动取像面前一面。
> - 脚本式蒙卡使用 `CriterionIndex=15` 时，Zemax 工具栏里的 `CriterionCompIndex` 下拉可能回读为“无”。这不代表补偿失败；实际补偿以 TSC 中的 `FASTFOCUS` / `OPTIMIZE n` / `OPTIMIZE n OD` 为准。可打开生成的 `.TSC` 检查是否包含对应优化行。

> **重要**：Worst / Best 在 Zemax API 是**同一个开关**（`IsSaveBestWorstUsed`），无法单独控制。程序逻辑是：两者任一为 Y 即开启保存，两者都为 N 才关闭。
> **保存数量 ≠ 逐项导出**：保存数量是「在结果里保留最差的前 N 个样本」，不是把每次运行各导出一个文件。逐项导出文件功能本期已移除。

## 五、程序内部运行逻辑（端到端）

```
入口（gui.py / tol_run.py）
   │  传入 zmx、config、outdir、connect
   ▼
pipeline.prepare_session()  ← GUI 与 CLI 共享
   ├─ excel_io.read_config()      读 Excel 各 输入_* sheet → cfg
   ├─ zos_connect.connect()       连接 Zemax（extension/standalone）
   ├─ zos_connect.open_as_copy()  另存工作副本（不动原文件）
   ├─ tde_builder.build_and_write()   向导 + 明细 → 写 TDE 公差表（模式≠无且补偿面>0 时追加 COMP）
   ├─ mfe_builder.build_and_save()    评价函数 sheet → 写 MFE → 存报告 .MF
   ├─ mfe_builder.build_comp_mf()     模式≠无时：GMTA 补偿专用 MF → 存 _comp.MF
   ├─ tsc_builder.build_and_write()   REPORT sheet → 生成 .TSC（模式≠无时走双 LOADMERIT）
   └─ sess.sys.Save()             保存副本
   ▼
pipeline.run_montecarlo(prep)
   ├─ make_runspec(prep)          按 run_params 生成 RunSpec
   └─ tol_runner.run()            配置 OpenTolerancing + 非阻塞 Run + 轮询
   ▼
ztd_reader.read_ztd()
   ├─ OpenToleranceDataViewer 读 MonteCarloData
   ├─ 优先从同名 TSC 的 REPORT 行反推列头
   └─ 导出统计 Excel：统计行 + MC001/MC002... 原始样本行
```

### 模块职责一览（`toltool/` 包）

| 模块 | 职责 |
|---|---|
| `pipeline.py` | **单一来源**：prepare_session + run_montecarlo，GUI 与 CLI 共享 |
| `zos_connect.py` | 连接 Zemax、另存副本、读镜头信息 |
| `excel_io.py` | 读配置、生成示例模板 |
| `tde_builder.py` | 向导 + 明细 → 写 TDE |
| `mfe_builder.py` | 评价函数 sheet → 写 MFE → 存 .MF |
| `tsc_builder.py` | REPORT sheet → 生成 .TSC（LOADMERIT + OPTIMIZE + 逐条 REPORT）|
| `tol_runner.py` | 配置并运行蒙卡（界面解耦：progress_cb 回调 + cancel_flag 取消）|
| `ztd_reader.py` | 读 ZTD MonteCarloData → 分项统计 |

### 文件调用顺序（一次完整运行）

```
入口文件（gui.py / tol_run.py）
   └─▶ toltool/pipeline.py : prepare_session()
          ├─▶ excel_io.py     read_config()        ① 读 Excel 各 输入_* sheet
          ├─▶ zos_connect.py  connect()/open_as_copy()  ② 连接 + 另存副本
          ├─▶ tde_builder.py  build_and_write()    ③ 写 TDE 公差表
          ├─▶ mfe_builder.py  build_and_save()     ④ 写 MFE → 存 .MF
          └─▶ tsc_builder.py  build_and_write()    ⑤ 生成 .TSC
   └─▶ toltool/pipeline.py : run_montecarlo()       ⑥ 跑蒙卡
          └─▶ tol_runner.py   run()                配置 OpenTolerancing + 跑蒙卡
   └─▶ toltool/ztd_reader.py : read_ztd()/export_excel()  ⑦ tol_run.py 默认；GUI 可自动/独立执行
```

> 一句话：**入口 → pipeline（依次调 excel_io / zos_connect / tde_builder / mfe_builder / tsc_builder）→ tol_runner 跑蒙卡 → ztd_reader 读取 ZTD、反推表头并导出统计 Excel**。`__init__.py` 仅作包标识，不参与流程逻辑。

## 六、输出产物

| 产物 | 位置 | 触发 |
|---|---|---|
| 工作副本 zmx | 输出目录 | 始终（不改原文件）|
| 评价函数 .MF | `<DataDir>\MeritFunction\` | 始终 |
| TSC 脚本 | `<DataDir>\Tolerance\` | 始终；ZTD 统计会用同名 TSC 反推 REPORT 表头 |
| 公差数据 .ZTD | 输出目录 | 始终 |
| Worst / Best Case | 由 Zemax 决定（运行后回读路径）| 保存WorstCase/BestCase 任一为 Y |
| 统计 Excel | ZTD 同目录，`<ZTD文件名>_统计.xlsx` | `输出统计Excel=Y` 或独立分析已有 ZTD；含自定义脚本、REPORT 分项和 TDE 中的 COMP 项 |

## 七、常见问题

- **连不上 / 连接报错**：多半是没进交互扩展，或上次连接已消耗等待状态 → 重新点一次「编程→交互扩展」。
- **写 TSC 报权限错误（PermissionError）**：用后台隐藏进程启动 python 会写不了 `Documents\Zemax\Tolerance`。**在你自己的终端前台运行即可**，这不是代码问题。
- **"运行中…" 只打印一次**：这是有意为之，蒙卡运行期间每 2 秒轮询一次，只在首次提示，避免刷屏。完成后会提示结果。
- **日志提示"补偿器模式未生效(实际索引 2=无)"**：脚本式蒙卡（User Script / `CriterionIndex=15`）下，Zemax 可能不接受工具栏补偿器下拉设置，回读仍显示“无”。这不代表补偿失败；实际补偿由 TSC 里的 `FASTFOCUS` / `OPTIMIZE n` / `OPTIMIZE n OD` 执行。真正决定是否补偿的是 Excel「补偿器模式」（=无则完全不补偿）。如需确认，可打开生成的 `.TSC`，检查是否有对应优化行。
- **公差分析失败但 GUI 日志很长**：程序会在运行前后打印 Zemax 公差工具关键字段，例如 `Succeeded`、`ErrorMessage`、`NumberToSave`、`IsSaveBestWorstUsed`、`SaveTolDataFile`、`TolDataFile`。这些字段用于判断是 Excel 配置、Zemax 变量/求解、Worst/Best 保存，还是 ZTD 保存阶段的问题。
- **统计 Excel 表头仍是 colN**：ZOS-API 的 `MonteCarloData.Values` 主要暴露数值矩阵，不一定能读到 GUI 表头。程序会按同名 `.TSC` 反推 REPORT 表头：优先查找 ZTD 同目录，其次查找当前 Zemax 数据目录下的 `Tolerance`，以及当前 Windows 用户 `Documents\Zemax\Tolerance` / OneDrive 文档目录。若仍找不到同名 TSC，会继续按 `colN` 生成 Excel，并在日志中提示已搜索路径。COMP 列不靠表头模糊匹配，而是依赖同目录 `run_config.json` 中记录的 TDE 顺序定位。
- **统计 Excel「名义值」行为空**：名义值在跑蒙特卡洛前从评价函数读取，写入 `run_config.json` 的 `report_meta`（带 `名义值来源=MFE_OPERAND`）。若该行全为空，日志会打印“REPORT 名义值全部为空”告警，通常是脚本式流程下 MFE `Value` 列尚未计算所致；此为运行时 API 行为，需在真实 Zemax 环境复跑确认。独立 ZTD 分析仅当同目录 `run_config.json` 含 `report_meta` 时才有名义值，旧结果目录（无该字段）不会回填。
- **改了 Excel 没效果**：确认填的是 `输入_*` sheet（不是 `示例_*`），且键名与上表一致。
- **GUI 进度为什么只有"运行总数/保存总数"而没有"第几次"**：脚本式蒙卡（TSC，CriterionIndex=15）运行期间 Zemax **不暴露任何逐次进度**——`Progress` 全程 0、结束才跳 100；MC 样本文件也是**跑完才一次性写**（实测），且只写 NumberToSave 个。TSC 的 `SAVE` 命令只能存固定一个文件、不支持变量，无法逐圈保存。因此 GUI 只能诚实显示总数 + 已用时间，做不到 1/N→N/N 的逐次进度。
- **想保留每次蒙卡样本做排查**：把「保存数量」填成与「蒙特卡洛次数」相等（如都填 200），跑完后输出目录下会有全部样本 zmx；代价是占磁盘、且仍是跑完才写。
